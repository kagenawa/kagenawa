Веду disform — частное агентство аудита продуктов на базе LLM. На сегодня —
единственная в РФ команда, специализирующаяся именно на угрозах LLM-слоя:
prompt injection, indirect injection через документы и RAG, jailbreak ролевых
рамок, утечки системных промптов, манипуляция инструментами агентов, отравление
векторных баз. Классические пентестеры этот слой пропускают. Зарубежные
аналоги — англоязычные и не учитывают 152-ФЗ.

Работа строится на трёх принципах: юридическая чистота каждого шага,
воспроизводимость каждой находки, инженерная честность каждого вердикта.

#### Подход

— Каждый аудит начинается с письменного согласия владельца, NDA и согласованного
scope. Без этого работа не начинается — статья 272 УК недвусмысленна.

— Полный аудит-трейл: audit_id, sha256(config), timestamps, оператор и payload
по каждой пробе. Любую находку можно поднять и воспроизвести спустя месяцы.

— Методология построена так, чтобы клиент не получал отчёт с 30% false-positive.
Каждая критичная находка проходит consensus-верификацию двумя независимыми
прогонами. Семантический классификатор сверяется с baseline по нейтральным
запросам. Snapshot before/after отлавливает случаи, когда продукт ломается
во время атаки и любая дальнейшая «находка» становится артефактом.

— Анти-халлюцинационный фильтр на уровне отчётности: whitelist восьми
поддерживаемых фреймворков плюс fact-check каждого упомянутого CVE по
локальному NVD-кешу. Если идентификатор не подтверждён — помечается
«требует уточнения», а не выдаётся за факт.

#### Spider

Собственный движок, на котором работает агентство.

LLM-уровень: 24+ векторов атак, multi-turn chains, indirect injection через
поднимаемый локально HTTP-сервер с подменными документами, агентные сценарии
для систем с tool-use и памятью, RAG-сценарии включая отравление через
загружаемые материалы.

Web-уровень: восемь модулей — TLS-конфиг и cipher suites, поиск утекших
ключей в клиентских бандлах, маппинг REST/GraphQL, fingerprinting фреймворков
с CVE fact-check, rate-limit и token-exhaustion аудит для LLM-эндпоинтов,
email header injection, опционально nuclei.

Адаптеры: REST, headless (Playwright), Telegram, GraphQL с детектором
auth-bypass на mutations. Под российский рынок — n8n, Dify, Flowise,
GigaChat, YandexGPT.

Pre-flight calibration: до запуска атак движок профилирует target на CAPTCHA,
WAF, rate-limit и оценивает context window. Если защита блокирует — клиент
узнаёт об этом сразу, а не через неделю по бесполезному отчёту.

Continuous monitoring: ежедневные прогоны по эндпоинтам клиента с алертами
в Telegram/SMTP при детекции новых уязвимостей.

152-ФЗ pre-check: шесть детекторов на политику конфиденциальности,
право на удаление, residency данных, аудит-логи, утечки ПДн через ответы LLM,
факт трансграничной передачи. Не юридическое заключение — техническая часть,
снимающая 80% типовых вопросов от регулятора.

#### Что мы не делаем

— Классический пентест без LLM-слоя. Это не наша ниша.
— ФСТЭК/ФСБ-сертификация. Лицензий нет, не вводим клиента в заблуждение.
— Готовые эксплойты и «боевые» инструменты атак — не разрабатываем,
   не продаём, не публикуем.
— Аудит без письменного согласия владельца. Не обсуждается.
